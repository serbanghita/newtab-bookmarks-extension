# New Tab Unlimited Bookmarks - Chrome Extension

Chromium extension (Manifest v3) for Google Chrome and Microsoft Edge that replaces the default New Tab page with a customizable bookmarks display. Published on the Chrome Web Store and Microsoft Edge Add-ons. The current version is tracked in `package.json` / `manifest.json`.

## Build & Dev

```bash
npm install          # install deps (devDependencies only: esbuild, typescript, eslint, @types/chrome, @puppeteer/browsers)
npm run dev          # esbuild watch mode with source maps
npm run build        # production bundle (dist/js/newtab.js)
npm run preview      # build + watch + launch Chrome for Testing with extension loaded
npm run lint         # ESLint over src/ (flat config in eslint.config.mjs)
```

No test suite exists (`npm test` is a no-op).

To test locally: `chrome://extensions/` (or `edge://extensions/` in Microsoft Edge) → Developer mode → Load unpacked → select project root.

## Architecture

Vanilla TypeScript, no framework. Zero runtime dependencies — only Chrome APIs + DOM.

**Entry point:** `src/index.ts` — initializes Settings → Bookmarks → View, then renders.

**Core classes (one per file in `src/`):**

- `Settings.ts` — reads/writes user prefs to `chrome.storage.local` (key: `"newtab-bookmarks"`). Uses typed enums (`BooleanSetting`, `SizeSetting`, `LayoutSetting`).
- `Bookmarks.ts` — wraps `chrome.bookmarks` API. Reads tree, supports CRUD, recursive search, multi-folder selection (comma-separated folder names).
- `View.ts` — all DOM rendering: bookmark items, search bar, settings dialog, drag-and-drop reordering. Largest file (~253 lines).
- `utils.ts` — DOM helpers (`$`, `$q`, `$$q`), `faviconURL()`, `determineFolderNames()`, `truncateLongText()`.
- `popup.ts` — stub (1 line).

**UI assets live in `dist/` (committed, not generated):**

- `dist/newtab/newtab.html` + CSS files — the new tab page
- `dist/popup/popup.html` + CSS — extension popup
- `dist/images/` — extension icons

Only `dist/js/newtab.js` is generated by the build.

## Releases & CHANGELOG

This project follows [Keep a Changelog 1.1.0](https://keepachangelog.com/en/1.1.0/) and SemVer.

Every release MUST update `CHANGELOG.md`. The flow for bumping from `x.y.z` to a new version:

1. Review changes since the last tag:
   `git log --pretty=format:'%h %s' v<previous>..HEAD`
2. In `CHANGELOG.md`, insert a new `## [<new-version>] - <YYYY-MM-DD>` heading immediately after the intro paragraph and before the previous version's heading. Use the tag/release date in ISO format (today, if cutting now).
3. Under the new version, group entries under `Added` / `Changed` / `Deprecated` / `Removed` / `Fixed` / `Security` — in that order, omitting any section with no entries. Write one short, user-facing line per change; do not paste raw commit subjects.
4. Add a new link reference at the bottom of `CHANGELOG.md`:
   `[<new-version>]: https://github.com/serbanghita/newtab-bookmarks-extension/compare/v<previous>...v<new-version>`
5. Bump `version` in both `package.json` and `manifest.json` to the new value.
6. Commit with subject `Bump version to <new-version>` (include the CHANGELOG, `package.json`, and `manifest.json` in the same commit).
7. Tag: `git tag v<new-version>` and `git push --tags`.
